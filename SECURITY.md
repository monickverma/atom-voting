# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes     |
| < 1.0   | ❌ No      |

## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a security issue, please report it responsibly by:

1. **Email**: Open a [GitHub Security Advisory](https://github.com/monickverma/atom-voting/security/advisories/new) (preferred), or email the maintainer directly.
2. **Include**:
   - A clear description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested mitigation (optional)

## Response Timeline

| Step | Time |
|------|------|
| Initial acknowledgement | Within **72 hours** |
| Assessment and status update | Within **7 days** |
| Security patch release | As soon as possible after assessment |

## Responsible Disclosure

We ask that you:

- Give us reasonable time to patch before public disclosure
- Not exploit the vulnerability beyond what is needed to demonstrate it
- Not access or modify other users' data

We commit to:

- Acknowledging your report promptly
- Keeping you informed of our progress
- Crediting you in the release notes (unless you prefer anonymity)

## Out of Scope

- Issues in dependencies (please report those to the dependency maintainer)
- Social engineering attacks
- Physical attacks on infrastructure
